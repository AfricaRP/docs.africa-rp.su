# Для редакторов

Здесь собраны все элементы форматирования, доступные при написании статей. Мы подготовили наглядные примеры по принципу «Как писать» -> «Как выглядит».

## Текст и клавиши

Вы можете использовать стандартное оформление текста, а также специальный тег `<Kbd>` для отображения клавиш клавиатуры.

**Как писать:**
```mdx
**Жирный текст**
*Курсив*
~~Зачеркнутый текст~~
`встроенный код`
Нажмите <Kbd>Alt</Kbd> + <Kbd>F4</Kbd>
[Ссылка на сайт](https://africa-rp.su)
```

**Как выглядит:**

**Жирный текст**
*Курсив*
~~Зачеркнутый текст~~
`встроенный код`
Нажмите <Kbd>Alt</Kbd> + <Kbd>F4</Kbd>
[Ссылка на сайт](https://africa-rp.su)

## Таблицы

Для создания таблиц используйте вертикальные черты `|` и дефисы `-`:

**Как писать:**
```markdown
| Колонка 1 | Колонка 2 | Колонка 3 |
| :--- | :---: | ---: |
| Левый край | По центру | Правый край |
| Текст | Текст | Текст |
```

**Как выглядит:**

| Колонка 1 | Колонка 2 | Колонка 3 |
| :--- | :---: | ---: |
| Левый край | По центру | Правый край |
| Текст | Текст | Текст |

## Видео, YouTube и RuTube

В документации можно использовать как локальные видео файлы (`<VideoPlayer>`), так и видео с YouTube (`<YouTube>`) или RuTube (`<Rutube>`).

**Как писать:**
```mdx
<VideoPlayer src="/media/tutorial/video.mp4" title="Тестовое видео" />

<YouTube id="fKABMFgUYXE" title="Официальный трейлер" />

<Rutube id="c0b30bb7e8e88c159eda91ac01fa9e14" title="Тестовое видео Rutube" />
```

**Как выглядит:**

<VideoPlayer src="/media/tutorial/video.mp4" title="Тестовое видео" />

<YouTube id="fKABMFgUYXE" title="Официальный трейлер" />

<Rutube id="c0b30bb7e8e88c159eda91ac01fa9e14" title="Тестовое видео Rutube" />

## Блоки внимания (Callouts)

Используйте `<Callout>` для выделения важной информации. Доступные типы (type): `note` (синий, по умолчанию), `tip` (зеленый), `warning` (желтый), `danger` (красный).

**Как писать:**
```mdx
<Callout type="tip" title="Полезный совет">
  Всегда пристегивайте ремень безопасности в машине!
</Callout>

<Callout type="danger" title="Важно">
  За использование читов выдается перманентный бан.
</Callout>
```

**Как выглядит:**

<Callout type="tip" title="Полезный совет">
  Всегда пристегивайте ремень безопасности в машине!
</Callout>

<Callout type="danger" title="Важно">
  За использование читов выдается перманентный бан.
</Callout>

## Многострочный код

Для создания блоков кода оберните текст в тройные обратные кавычки и укажите язык.
Чтобы добавить красивый заголовок с названием файла, напишите `имя_файла.js` сразу после языка.

**Как писать:**
````markdown
```javascript app.js
console.log("Привет, AfricaRP!");
```
````

**Как выглядит:**

```javascript app.js
console.log("Привет, AfricaRP!");
```

## Подсветка строк кода (!mark)

Используйте комментарий `// !mark` в конце строки, чтобы подсветить её синим фоном и выделить на фоне остального кода.
*(Обратите внимание: в блоке "Как писать" мы добавили пробел после знака восклицания, чтобы код отобразился как текст. В реальном использовании пишите слитно: `// !mark`)*

**Как писать:**
````markdown
```javascript server.js
const express = require('express');
const app = express(); // ! mark

app.listen(3000);
```
````

**Как выглядит:**

```javascript server.js
const express = require('express');
const app = express(); // !mark

app.listen(3000);
```

## Фокус на строках (!focus)

Если кода много, вы можете сфокусировать внимание зрителя только на нужной строке с помощью комментария `// !focus`. Эта строка будет выделена серым фоном, а все остальные строки в блоке будут размыты и затемнены.

**Как писать:**
````markdown
```javascript server.js
const express = require('express'); // ! focus
const app = express();

app.listen(3000);
```
````

**Как выглядит:**

```javascript server.js
const express = require('express'); // !focus
const app = express();

app.listen(3000);
```

## Пошаговый процесс (Steps)

Используйте `<Steps>` для создания стильных пошаговых инструкций (таймлайнов). Внутри оборачивайте каждый шаг в обычный заголовок `###`.

**Как писать:**
```mdx
<Steps>
### Скачайте лаунчер
Перейдите на главную страницу и нажмите кнопку "Скачать".

### Установите моды
Откройте настройки и выберите нужные модификации.

### Заходите на сервер
Всё готово, приятной игры!
</Steps>
```

**Как выглядит:**

<Steps>
### Скачайте лаунчер
Перейдите на главную страницу и нажмите кнопку "Скачать".

### Установите моды

## Раскрывающиеся блоки (Accordion)

Используйте `<Accordion>` для скрытия длинных текстов, FAQ или деталей, которые не нужны каждому читателю с первого взгляда. Заголовок аккордеона задается через параметр `title`.

**Как писать:**
```mdx
<Accordion title="Как получить права администратора?">
  Для этого необходимо оставить заявку на форуме и пройти собеседование.
</Accordion>
```

**Как выглядит:**

<Accordion title="Как получить права администратора?">
  Для этого необходимо оставить заявку на форуме и пройти собеседование.
</Accordion>

## Бейджи статусов (Badge)

Стильные ярлыки `<Badge>`, которые можно вставлять в текст или заголовки. Доступные типы (`type`): `default`, `new`, `deprecated`, `beta`.

**Как писать:**
```mdx
Добавлен новый автомобиль <Badge type="new">Новое</Badge>

Старая команда `/spawn` больше не работает <Badge type="deprecated">Устарело</Badge>
```

**Как выглядит:**

Добавлен новый автомобиль <Badge type="new">Новое</Badge>

Старая команда `/spawn` больше не работает <Badge type="deprecated">Устарело</Badge>

**Как выглядит:**

## Цитаты и Буквица

Специальное оформление для лора, правил или художественных вставок. `<DropCap>` увеличивает первую букву, а `<Quote>` создает стилизованный блок цитаты.

**Как писать:**
```mdx
<Quote author="Главная администрация">
  <DropCap>A</DropCap>fricaRP — это место, где история пишется самими игроками.
  Каждый ваш шаг меняет судьбу сервера.
</Quote>
```

**Как выглядит:**

<Quote author="Главная администрация">
  <DropCap>A</DropCap>fricaRP — это место, где история пишется самими игроками.

</Quote>

## Правильно / Неправильно (Do/Dont)

Используйте сетку `<DoDontGrid>` и блоки `<DoDont>` для наглядного сравнения правильного и ошибочного подходов. Отлично сочетается с математическими формулами в формате LaTeX (`$$`).

**Как писать:**
```mdx
<DoDontGrid>
  <DoDont type="do">
    Интегральная формула Коши для $n$-й производной аналитической функции в ТФКП (с учетом факториала и правильной степени):
    $$
    f^{(n)}(z_0) = \frac{n!}{2\pi i} \oint_{\gamma} \frac{f(z)}{(z - z_0)^{n+1}} \, dz
    $$
  </DoDont>
  <DoDont type="dont">
    Грубая ошибка студентов на экзамене по Вышмату — потеря факториала и смещение степени в знаменателе:
    $$
    f^{(n)}(z_0) = \frac{1}{2\pi i} \oint_{\gamma} \frac{f(z)}{(z - z_0)^n} \, dz
    $$
  </DoDont>
</DoDontGrid>
```

**Как выглядит:**

<DoDontGrid>
  <DoDont type="do">
    Интегральная формула Коши для $n$-й производной аналитической функции в ТФКП (с учетом факториала и правильной степени):
    $$
    f^{(n)}(z_0) = \frac{n!}{2\pi i} \oint_{\gamma} \frac{f(z)}{(z - z_0)^{n+1}} \, dz
    $$
  </DoDont>
  <DoDont type="dont">
    Грубая ошибка студентов на экзамене по Вышмату — потеря факториала и смещение степени в знаменателе:
    $$
    f^{(n)}(z_0) = \frac{1}{2\pi i} \oint_{\gamma} \frac{f(z)}{(z - z_0)^n} \, dz

  </DoDont>
</DoDontGrid>

```mdx
    Получить одобрение от старшей администрации (Заблокировано до прохождения обзвона)
  </ChecklistItem>
</Checklist>
```

## Упоминания ролей (Mention)

Тег `<Mention>` создает красивую плашку, похожую на пинг (упоминание) роли в Discord. Доступные роли (`role`): `admin`, `moderator`, `developer`, `vip`, `user`.

**Как писать:**
```mdx
Если вы нашли баг, сообщите об этом <Mention role="developer">Разработчикам</Mention>.
Для снятия наказания обращайтесь к <Mention role="admin">Главной администрации</Mention> или <Mention role="moderator">Модераторам</Mention>.
```

**Как выглядит:**

Если вы нашли баг, сообщите об этом <Mention role="developer">Разработчикам</Mention>.
Для снятия наказания обращайтесь к <Mention role="admin">Главной администрации</Mention> или <Mention role="moderator">Модераторам</Mention>.

## Секретный текст (Secret)

Компонент `<Secret>` заблюривает текст, создавая интригу. Содержимое становится читаемым только при наведении курсора.

**Как писать:**
```mdx
Главное правило бойцовского клуба: <Secret>никому не рассказывать о бойцовском клубе</Secret>.
```

**Как выглядит:**

Главное правило бойцовского клуба: <Secret>никому не рассказывать о бойцовском клубе</Secret>.

## Дерево решений (DecisionTree)

Интерактивный компонент для создания пошаговых опросов или алгоритмов устранения неполадок.

**Как писать:**
```mdx
<DecisionTree initial="start">
  <DecisionStep step="start">
    ### 1. Вы используете Windows или macOS?
    <DecisionButton target="windows">Windows</DecisionButton>
    <DecisionButton target="macos">macOS</DecisionButton>
  </DecisionStep>
  
  <DecisionStep step="windows">
    ### 2. Загрузите файл .exe
    Убедитесь, что у вас есть права администратора.
    <DecisionButton target="finish">Я скачал</DecisionButton>
    <DecisionButton target="start">Назад</DecisionButton>
  </DecisionStep>

  <DecisionStep step="macos">
    ### 2. Загрузите файл .dmg
    Если система блокирует установку, перейдите в настройки безопасности.
    <DecisionButton target="finish">Я скачал</DecisionButton>
    <DecisionButton target="start">Назад</DecisionButton>
  </DecisionStep>

  <DecisionStep step="finish">
    ### 3. Установка завершена!
    Теперь вы можете открыть приложение.
    <DecisionButton target="start">Начать заново</DecisionButton>
  </DecisionStep>
</DecisionTree>
```

**Как выглядит:**

## Встроенный аудиоплеер (VoiceBox)

Используйте `<VoiceBox>`, чтобы встроить элегантный плеер прямо в статью. Идеально для прослушивания голосовых логов, произношения терминов или атмосферных саундтреков.

**Как писать:**
```mdx
<VoiceBox src="/media/tutorial/sound.mp3" title="Шеф выдал базу" />
```

**Как выглядит:**

<VoiceBox src="/media/tutorial/sound.mp3" title="Шеф выдал базу" />
