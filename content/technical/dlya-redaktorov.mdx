# Для редакторов

Здесь собраны все элементы форматирования, доступные при написании статей. Мы подготовили наглядные примеры по принципу «Как писать» -> «Как выглядит».

## Текст и клавиши

Вы можете использовать стандартное оформление текста, а также специальный тег `<Kbd>` для отображения клавиш клавиатуры.

**Как писать:**
```mdx
**Жирный текст**
*Курсив*
~~Зачеркнутый текст~~
`встроенный код`
Нажмите <Kbd>Alt</Kbd> + <Kbd>F4</Kbd>
[Ссылка на сайт](https://africa-rp.su)
```

**Как выглядит:**

**Жирный текст**
*Курсив*
~~Зачеркнутый текст~~
`встроенный код`
Нажмите <Kbd>Alt</Kbd> + <Kbd>F4</Kbd>
[Ссылка на сайт](https://africa-rp.su)

```markdown
Нумерованный список:
```

## Таблицы

Для создания таблиц используйте вертикальные черты `|` и дефисы `-`:

**Как писать:**
```markdown

| :--- | :---: | ---: |
| Левый край | По центру | Правый край |
| Текст | Текст | Текст |
```

**Как выглядит:**

| :--- | :---: | ---: |
| Левый край | По центру | Правый край |

## Слайдер глубины погружения

Если тема сложная, дайте читателю выбрать уровень детализации с помощью `<DepthSection>`. Слайдер имеет три уровня: TL;DR (кратко), Стандарт и Deep Dive (глубокое погружение).

**Как писать:**
```mdx
<DepthSection>
  <DepthLevel showAt="tldr">
    **TL;DR:** База данных хранит информацию пользователей.
  </DepthLevel>
  <DepthLevel showAt="standard">
    Мы используем реляционную базу данных PostgreSQL. Она позволяет связывать таблицы пользователей, платежей и сессий через внешние ключи.
  </DepthLevel>
  <DepthLevel showAt="deep">
    На уровне движка мы настроили кастомные индексы B-Tree для ускорения поиска по UUID, а также партицирование логов по месяцам, что снизило нагрузку на диск на 30%.
  </DepthLevel>
</DepthSection>
```

**Как выглядит:**
<DepthSection>
  <DepthLevel showAt="tldr">

## Умная матрица сравнения

Компонент `<CompareMatrix>` оборачивает обычную Markdown-таблицу. Он закрепляет первую колонку при скролле на мобильных устройствах и автоматически заменяет плюсы, минусы, "Да" и "Нет" на красивые иконки.

**Как писать:**
```mdx
<CompareMatrix>
| Фича | SQL | NoSQL |
| --- | --- | --- |
| Жесткая схема | + | - |
| Масштабирование | Вертикальное | Горизонтальное |
| Транзакции (ACID) | Да | Нет |
</CompareMatrix>
```

**Как выглядит:**
<CompareMatrix>
| Фича | SQL | NoSQL |
| --- | --- | --- |
| Жесткая схема | + | - |
| Масштабирование | Вертикальное | Горизонтальное |
| Транзакции (ACID) | Да | Нет |
</CompareMatrix>

## Объяснение кода (CodeWalkthrough)

Компонент `<CodeWalkthrough>` создает layout, где левая колонка предназначена для текста, а правая — для блока кода. Код прилипает к верху экрана при скролле.

**Как писать:**
````mdx
<CodeWalkthrough>
  <WalkthroughText>
    Сначала мы импортируем необходимые модули.
    
    Затем создаем функцию `init()`, которая устанавливает соединение.
    
    В конце не забываем обработать ошибки с помощью блока `catch`.
  </WalkthroughText>
  <WalkthroughCode>

```javascript
import { db } from './database';

async function init() {
  try {
    await db.connect();
  } catch (e) {
    console.error(e);
  }
}
```

  </WalkthroughCode>
</CodeWalkthrough>
````

**Как выглядит:**
<CodeWalkthrough>
  <WalkthroughText>
    Сначала мы импортируем необходимые модули.
    
    Затем создаем функцию `init()`, которая устанавливает соединение.
    
    В конце не забываем обработать ошибки с помощью блока `catch`.
  </WalkthroughText>
  <WalkthroughCode>

```javascript
import { db } from './database';

async function init() {
  try {
    await db.connect();
  } catch (e) {
    console.error(e);
  }
}
```

  </WalkthroughCode>
</CodeWalkthrough>

## Подсветка строк кода (!mark)

Используйте комментарий `// !mark` в конце строки, чтобы подсветить её синим фоном и выделить на фоне остального кода.
*(Обратите внимание: в блоке "Как писать" мы добавили пробел после знака восклицания, чтобы код отобразился как текст. В реальном использовании пишите слитно: `// !mark`)*

**Как писать:**
````markdown
```javascript server.js
const express = require('express');
const app = express(); // ! mark

app.listen(3000);
```
````

**Как выглядит:**

```javascript server.js
const express = require('express');
const app = express(); // !mark

app.listen(3000);
```

## Фокус на строках (!focus)

Если кода много, вы можете сфокусировать внимание зрителя только на нужной строке с помощью комментария `// !focus`. Эта строка будет выделена серым фоном, а все остальные строки в блоке будут размыты и затемнены.

**Как писать:**
````markdown
```javascript server.js
const express = require('express'); // ! focus
const app = express();

app.listen(3000);
```
````

**Как выглядит:**

```javascript server.js

```

## Раскрывающиеся блоки (Accordion)

```mdx
<Accordion title="Как получить права администратора?">
```

## Бейджи статусов (Badge)

Стильные ярлыки `<Badge>`, которые можно вставлять в текст или заголовки. Доступные типы (`type`): `default`, `new`, `deprecated`, `beta`.

**Как писать:**
```mdx
Добавлен новый автомобиль <Badge type="new">Новое</Badge>

Старая команда `/spawn` больше не работает <Badge type="deprecated">Устарело</Badge>
```

**Как выглядит:**

Добавлен новый автомобиль <Badge type="new">Новое</Badge>

Старая команда `/spawn` больше не работает <Badge type="deprecated">Устарело</Badge>

## Правильно / Неправильно (Do/Dont)

Используйте сетку `<DoDontGrid>` и блоки `<DoDont>` для наглядного сравнения правильного и ошибочного подходов. Отлично сочетается с математическими формулами в формате LaTeX (`$$`).

**Как писать:**
```mdx
<DoDontGrid>

    $$
    f^{(n)}(z_0) = \frac{n!}{2\pi i} \oint_{\gamma} \frac{f(z)}{(z - z_0)^{n+1}} \, dz
    $$
  </DoDont>
  <DoDont type="dont">
    Грубая ошибка студентов на экзамене по Вышмату — потеря факториала и смещение степени в знаменателе:
    $$
    f^{(n)}(z_0) = \frac{1}{2\pi i} \oint_{\gamma} \frac{f(z)}{(z - z_0)^n} \, dz
    $$
  </DoDont>
</DoDontGrid>
```

**Как выглядит:**

<DoDontGrid>
  <DoDont type="do">
